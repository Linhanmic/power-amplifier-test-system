# 数据导入AI提示词

将Excel历史数据导入到SQLite数据库的AI提示词方案。

---

## 提示词1：分析Excel结构

```
我需要将Excel数据导入到SQLite数据库中。请帮我分析以下Excel文件的结构：

1. 文件路径：[填写Excel文件路径]
2. 工作表名称：[填写工作表名称]

请完成以下分析：
1. 识别所有列名及其数据类型
2. 识别数据范围（有多少行数据）
3. 识别是否存在合并单元格
4. 识别数据之间的关联关系（如车型列、需求ID等）
5. 输出一份数据结构摘要

然后根据分析结果，生成对应的SQLite建表语句（CREATE TABLE），要求：
- 字段类型使用SQLite支持的类型（TEXT, INTEGER, REAL, BLOB）
- 设置合适的主键和索引
- 添加created_at和updated_at时间戳字段
```

## 提示词2：创建数据库和表结构

```
请帮我创建SQLite数据库，包含以下表结构。

数据库路径：[填写数据库路径，如 ./data/test_system.db]

需要创建的表：

1. **platforms** - 平台表
   - id: INTEGER PRIMARY KEY AUTOINCREMENT
   - platform_code: TEXT UNIQUE NOT NULL -- 平台代码
   - platform_name: TEXT NOT NULL -- 平台名称
   - description: TEXT -- 描述
   - created_at: TIMESTAMP DEFAULT CURRENT_TIMESTAMP
   - updated_at: TIMESTAMP DEFAULT CURRENT_TIMESTAMP

2. **vehicle_models** - 车型表
   - id: INTEGER PRIMARY KEY AUTOINCREMENT
   - model_code: TEXT UNIQUE NOT NULL -- 车型代码
   - model_name: TEXT NOT NULL -- 车型名称
   - platform_id: INTEGER NOT NULL -- 所属平台(FK)
   - year: INTEGER -- 年款
   - trim_level: TEXT -- 配置级别
   - description: TEXT
   - is_active: BOOLEAN DEFAULT 1
   - created_at, updated_at

3. **requirements** - 需求分析表
   - id: INTEGER PRIMARY KEY AUTOINCREMENT
   - req_id: TEXT UNIQUE NOT NULL -- 需求编号
   - title: TEXT NOT NULL -- 需求标题
   - description: TEXT
   - category: TEXT -- 需求分类
   - priority: TEXT CHECK(priority IN ('high','medium','low'))
   - status: TEXT CHECK(status IN ('draft','approved','implemented','verified'))
   - source: TEXT -- 需求来源
   - created_at, updated_at

4. **requirement_vehicle_details** - 需求-车型关联表
   - id: INTEGER PRIMARY KEY AUTOINCREMENT
   - requirement_id: INTEGER NOT NULL -- FK to requirements
   - vehicle_model_id: INTEGER NOT NULL -- FK to vehicle_models
   - is_applicable: BOOLEAN DEFAULT 1
   - implementation_detail: TEXT
   - specific_parameter: TEXT
   - notes: TEXT
   - status: TEXT CHECK(status IN ('pending','confirmed','excluded'))
   - created_at, updated_at
   - UNIQUE(requirement_id, vehicle_model_id)

5. **test_case_groups** - 测试用例分组表
   - id: INTEGER PRIMARY KEY AUTOINCREMENT
   - group_name: TEXT NOT NULL
   - parent_id: INTEGER -- 自引用FK
   - sort_order: INTEGER DEFAULT 0
   - description: TEXT
   - created_at, updated_at

6. **test_cases** - 测试用例表
   - id: INTEGER PRIMARY KEY AUTOINCREMENT
   - case_id: TEXT UNIQUE NOT NULL -- 用例编号
   - group_id: INTEGER -- FK to test_case_groups
   - scenario_id: INTEGER -- FK to test_scenarios
   - data_row_index: INTEGER
   - requirement_id: INTEGER -- FK to requirements
   - preconditions: TEXT
   - steps: TEXT
   - expected_result: TEXT
   - priority: TEXT CHECK(priority IN ('high','medium','low'))
   - author: TEXT
   - status: TEXT CHECK(status IN ('draft','approved','automated','deprecated'))
   - created_at, updated_at

7. **test_case_vehicles** - 测试用例-车型关联表
   - id: INTEGER PRIMARY KEY AUTOINCREMENT
   - test_case_id: INTEGER NOT NULL -- FK to test_cases
   - vehicle_model_id: INTEGER NOT NULL -- FK to vehicle_models
   - specific_note: TEXT
   - created_at
   - UNIQUE(test_case_id, vehicle_model_id)

8. **test_scripts** - 测试脚本表(Gauge spec文件)
   - id: INTEGER PRIMARY KEY AUTOINCREMENT
   - script_id: TEXT UNIQUE NOT NULL
   - spec_name: TEXT NOT NULL
   - file_path: TEXT
   - description: TEXT
   - version: TEXT
   - status: TEXT CHECK(status IN ('active','inactive','deprecated'))
   - author: TEXT
   - created_at, updated_at

9. **test_scenarios** - 测试场景表(Gauge scenario)
   - id: INTEGER PRIMARY KEY AUTOINCREMENT
   - script_id: INTEGER NOT NULL -- FK to test_scripts
   - scenario_name: TEXT NOT NULL
   - sort_order: INTEGER DEFAULT 0
   - is_data_driven: BOOLEAN DEFAULT 0
   - data_table: TEXT -- JSON格式
   - created_at, updated_at

10. **can_matrices** - CAN通讯矩阵表
    - id: INTEGER PRIMARY KEY AUTOINCREMENT
    - matrix_id: TEXT UNIQUE NOT NULL
    - name: TEXT NOT NULL
    - version: TEXT
    - description: TEXT
    - created_at, updated_at

11. **signal_definitions** - 信号定义表
    - id: INTEGER PRIMARY KEY AUTOINCREMENT
    - matrix_id: INTEGER -- FK to can_matrices
    - signal_name: TEXT NOT NULL
    - message_id: TEXT
    - sub_id: TEXT -- 子ID
    - start_bit: INTEGER
    - length: INTEGER
    - byte_order: TEXT CHECK(byte_order IN ('little_endian','big_endian'))
    - value_type: TEXT CHECK(value_type IN ('unsigned','signed'))
    - factor: REAL
    - offset: REAL
    - min_value: REAL
    - max_value: REAL
    - unit: TEXT
    - value_table: TEXT -- JSON格式，如{"0":"关闭","1":"开启"}
    - description: TEXT
    - created_at

12. **vehicle_configs** - 车型配置表
    - id: INTEGER PRIMARY KEY AUTOINCREMENT
    - config_id: TEXT UNIQUE NOT NULL
    - vehicle_model: TEXT NOT NULL
    - year: INTEGER
    - trim_level: TEXT
    - description: TEXT
    - created_at, updated_at

13. **speaker_channel_mappings** - 扬声器通道映射表
    - id: INTEGER PRIMARY KEY AUTOINCREMENT
    - config_id: INTEGER -- FK to vehicle_configs
    - channel_number: INTEGER NOT NULL
    - speaker_position: TEXT NOT NULL
    - description: TEXT
    - created_at

14. **a2b_slots** - A2B SLOT表
    - id: INTEGER PRIMARY KEY AUTOINCREMENT
    - slot_number: INTEGER UNIQUE NOT NULL
    - slot_name: TEXT
    - max_channels: INTEGER
    - description: TEXT
    - created_at

15. **audio_source_types** - 音源类型表
    - id: INTEGER PRIMARY KEY AUTOINCREMENT
    - source_code: TEXT UNIQUE NOT NULL
    - source_name: TEXT NOT NULL
    - channel_count: INTEGER NOT NULL
    - description: TEXT
    - created_at, updated_at

16. **audio_source_slot_mappings** - 音源-SLOT映射表
    - id: INTEGER PRIMARY KEY AUTOINCREMENT
    - source_type_id: INTEGER NOT NULL -- FK to audio_source_types
    - slot_id: INTEGER NOT NULL -- FK to a2b_slots
    - slot_role: TEXT
    - sort_order: INTEGER DEFAULT 0
    - created_at
    - UNIQUE(source_type_id, slot_id)

17. **playback_matrices** - 播放矩阵主表
    - id: INTEGER PRIMARY KEY AUTOINCREMENT
    - matrix_id: TEXT UNIQUE NOT NULL
    - name: TEXT NOT NULL
    - config_id: INTEGER NOT NULL -- FK to vehicle_configs
    - description: TEXT
    - created_at, updated_at

18. **playback_matrix_base** - 基本播放矩阵表
    - id: INTEGER PRIMARY KEY AUTOINCREMENT
    - matrix_id: INTEGER NOT NULL -- FK to playback_matrices
    - source_slot_mapping_id: INTEGER NOT NULL -- FK to audio_source_slot_mappings
    - speaker_mapping: TEXT -- JSON数组，如[1,3,5]
    - description: TEXT
    - created_at, updated_at
    - UNIQUE(matrix_id, source_slot_mapping_id)

19. **playback_matrix_conditions** - 工况播放矩阵表
    - id: INTEGER PRIMARY KEY AUTOINCREMENT
    - matrix_id: INTEGER NOT NULL -- FK to playback_matrices
    - source_type_id: INTEGER NOT NULL -- FK to audio_source_types
    - source_status: TEXT
    - slot_ids: TEXT NOT NULL -- JSON数组，如[1,2]
    - headrest_mode: TEXT
    - sound_field_focus: TEXT
    - speaker_mapping: TEXT -- JSON数组
    - description: TEXT
    - created_at, updated_at

请生成完整的SQL脚本，包含：
1. 创建数据库目录（如果不存在）
2. 所有表的CREATE TABLE语句
3. 所有外键约束
4. 必要的索引
```

## 提示词3：Excel数据导入SQLite

```
我需要将Excel数据导入到刚才创建的SQLite数据库中。

Excel文件信息：
- 文件路径：[填写路径]
- 工作表：[填写工作表名称]
- 数据起始行：[如第2行，第1行为表头]
- 数据列映射：
  [根据提示词1的分析结果填写，例如：
   A列 → platform_code
   B列 → platform_name
   ...]

请编写Python脚本完成以下任务：

1. 使用openpyxl或pandas读取Excel文件
2. 数据清洗和转换：
   - 处理空值
   - 转换数据类型（字符串→整数等）
   - 处理特殊格式（如"是/否"→1/0）
3. 数据验证：
   - 检查必填字段
   - 检查外键关联是否存在
   - 检查唯一性约束
4. 批量插入数据：
   - 使用事务确保数据一致性
   - 处理重复数据（跳过或更新）
   - 记录导入日志
5. 输出导入结果统计：
   - 成功导入条数
   - 跳过条数
   - 失败条数及原因

注意事项：
- Excel中车型列可能有多个车型（如"Model-A, Model-B"），需要拆分后关联
- 分组信息可能需要解析层级关系（如"A/B/C"表示三级分组）
- JSON字段需要正确格式化（如speaker_mapping, slot_ids）

请提供完整的Python脚本代码。
```

## 提示词4：批量导入所有Excel

```
我有以下Excel文件需要批量导入到SQLite数据库：

1. 需求分析表：[路径]
2. 测试用例：[路径]
3. CAN通讯矩阵：[路径]
4. 扬声器通道映射：[路径]
5. 播放矩阵：[路径]

请编写一个统一的导入脚本，要求：

1. 按照依赖关系顺序导入：
   - 先导入平台和车型（基础数据）
   - 再导入需求、用例等业务数据
   - 最后导入关联数据

2. 每个Excel文件的导入逻辑封装成独立函数

3. 统一的错误处理和日志记录

4. 导入完成后生成数据质量报告：
   - 各表数据量统计
   - 关联完整性检查
   - 数据一致性校验

5. 支持断点续传（记录导入进度，失败后可从断点继续）

请提供完整的Python脚本。
```

---

## 使用步骤

1. **第一步**：用提示词1分析每个Excel文件的结构
2. **第二步**：用提示词2创建SQLite数据库和表
3. **第三步**：根据分析结果，用提示词3逐个导入Excel数据
4. **第四步**：（可选）用提示词4编写批量导入脚本
