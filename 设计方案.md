# 功放测试管理系统设计方案

## 项目概述

本系统用于管理功放（功率放大器）测试相关的全流程数据，包括需求分析、测试用例、测试脚本、CAN通讯矩阵、车型配置、扬声器通道映射和播放矩阵等模块。系统各模块之间存在紧密的数据关联关系。

## 技术架构

### 技术栈选择
- **前端**: Vue 3 + Element Plus + Vite
- **后端**: Flask + SQLAlchemy + Flask-RESTful
- **数据库**: SQLite（小团队适用，易于部署和维护）
- **测试框架**: Gauge（脚本管理，不含执行）
- **部署**: Docker容器化部署

### 数据库表结构

#### 1. 平台表 (platforms)
```sql
CREATE TABLE platforms (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    platform_code VARCHAR(50) UNIQUE NOT NULL, -- 平台代码
    platform_name VARCHAR(100) NOT NULL,       -- 平台名称
    description TEXT,                          -- 平台描述
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### 2. 车型表 (vehicle_models)
```sql
CREATE TABLE vehicle_models (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    model_code VARCHAR(50) UNIQUE NOT NULL,    -- 车型代码
    model_name VARCHAR(100) NOT NULL,          -- 车型名称
    platform_id INTEGER NOT NULL,              -- 所属平台
    year INTEGER,                              -- 年款
    trim_level VARCHAR(50),                    -- 配置级别
    description TEXT,                          -- 车型描述
    is_active BOOLEAN DEFAULT TRUE,            -- 是否启用
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (platform_id) REFERENCES platforms(id)
);
```

#### 3. 需求分析表 (requirements)
```sql
CREATE TABLE requirements (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    req_id VARCHAR(50) UNIQUE NOT NULL,        -- 需求编号
    title VARCHAR(200) NOT NULL,               -- 需求标题
    description TEXT,                          -- 需求描述
    category VARCHAR(100),                     -- 需求分类
    priority ENUM('high', 'medium', 'low'),   -- 优先级
    status ENUM('draft', 'approved', 'implemented', 'verified'), -- 状态
    source VARCHAR(100),                       -- 需求来源
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### 4. 需求-车型关联表 (requirement_vehicle_details)
```sql
-- 存储每个需求在每个车型上的具体细节
CREATE TABLE requirement_vehicle_details (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    requirement_id INTEGER NOT NULL,           -- 关联需求
    vehicle_model_id INTEGER NOT NULL,         -- 关联车型
    is_applicable BOOLEAN DEFAULT TRUE,        -- 是否适用
    implementation_detail TEXT,                -- 该车型的实现细节
    specific_parameter TEXT,                   -- 车型特定参数
    notes TEXT,                                -- 备注
    status ENUM('pending', 'confirmed', 'excluded'), -- 状态
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (requirement_id) REFERENCES requirements(id) ON DELETE CASCADE,
    FOREIGN KEY (vehicle_model_id) REFERENCES vehicle_models(id) ON DELETE CASCADE,
    UNIQUE(requirement_id, vehicle_model_id)   -- 同一需求同一车型只能有一条记录
);
```

#### 5. 测试用例分组表 (test_case_groups)
```sql
-- 支持多级分组的树形结构
CREATE TABLE test_case_groups (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    group_name VARCHAR(200) NOT NULL,          -- 分组名称
    parent_id INTEGER,                         -- 父分组ID（NULL表示顶级分组）
    sort_order INTEGER DEFAULT 0,              -- 排序序号
    description TEXT,                          -- 分组描述
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (parent_id) REFERENCES test_case_groups(id) ON DELETE CASCADE
);
```

#### 6. 测试用例表 (test_cases)
```sql
CREATE TABLE test_cases (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    case_id VARCHAR(50) UNIQUE NOT NULL,       -- 用例ID
    group_id INTEGER,                          -- 所属分组
    scenario_id INTEGER,                       -- 所属场景（Gauge scenario）
    data_row_index INTEGER,                    -- 数据驱动场景中的数据行索引
    requirement_id INTEGER,                    -- 上游需求ID
    preconditions TEXT,                        -- 前置条件
    steps TEXT,                                -- 测试步骤
    expected_result TEXT,                      -- 预期结果
    priority ENUM('high', 'medium', 'low'),   -- 优先级
    author VARCHAR(100),                       -- 编制人
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP, -- 编制日期
    status ENUM('draft', 'approved', 'automated', 'deprecated'), -- 状态
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (group_id) REFERENCES test_case_groups(id) ON DELETE SET NULL,
    FOREIGN KEY (scenario_id) REFERENCES test_scenarios(id) ON DELETE SET NULL,
    FOREIGN KEY (requirement_id) REFERENCES requirements(id)
);
```

#### 6. 测试用例-车型关联表 (test_case_vehicles)
```sql
-- 存储测试用例适用的车型
CREATE TABLE test_case_vehicles (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    test_case_id INTEGER NOT NULL,             -- 关联测试用例
    vehicle_model_id INTEGER NOT NULL,         -- 关联车型
    specific_note TEXT,                        -- 车型特定说明
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (test_case_id) REFERENCES test_cases(id) ON DELETE CASCADE,
    FOREIGN KEY (vehicle_model_id) REFERENCES vehicle_models(id) ON DELETE CASCADE,
    UNIQUE(test_case_id, vehicle_model_id)     -- 同一用例同一车型只能有一条记录
);
```

#### 8. 测试脚本表 (test_scripts) - 对应Gauge的spec文件
```sql
CREATE TABLE test_scripts (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    script_id VARCHAR(50) UNIQUE NOT NULL,     -- 脚本编号
    spec_name VARCHAR(200) NOT NULL,           -- spec文件名
    file_path VARCHAR(500),                    -- spec文件路径
    description TEXT,                          -- 脚本描述（spec文件注释）
    version VARCHAR(20),                       -- 版本号
    status ENUM('active', 'inactive', 'deprecated'), -- 状态
    author VARCHAR(100),                       -- 编写人
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### 9. 测试场景表 (test_scenarios) - 对应Gauge的scenario
```sql
-- 一个spec可包含多个scenario
-- 非数据驱动：一个scenario对应一个测试用例
-- 数据驱动：一个scenario对应多个测试用例（每个数据行一个用例）
-- 适用车型通过关联的测试用例获取（test_case_vehicles表）
CREATE TABLE test_scenarios (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    script_id INTEGER NOT NULL,                -- 所属脚本（spec文件）
    scenario_name VARCHAR(200) NOT NULL,       -- 场景名称
    sort_order INTEGER DEFAULT 0,              -- 场景在spec中的顺序
    is_data_driven BOOLEAN DEFAULT FALSE,      -- 是否数据驱动（Table驱动）
    data_table TEXT,                           -- 数据驱动参数表（JSON格式）
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (script_id) REFERENCES test_scripts(id) ON DELETE CASCADE
);
```

#### 5. 信号定义表 (signal_definitions)
```sql
CREATE TABLE signal_definitions (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    matrix_id INTEGER,                         -- 所属矩阵
    signal_name VARCHAR(100) NOT NULL,         -- 信号名称
    message_id VARCHAR(20),                    -- 报文ID
    sub_id VARCHAR(50),                        -- 子ID
    start_bit INTEGER,                         -- 起始位
    length INTEGER,                            -- 信号长度
    byte_order ENUM('little_endian', 'big_endian'), -- 字节序
    value_type ENUM('unsigned', 'signed'),     -- 值类型
    factor DECIMAL(10,4),                      -- 系数
    offset DECIMAL(10,4),                      -- 偏移量
    min_value DECIMAL(10,4),                   -- 最小值
    max_value DECIMAL(10,4),                   -- 最大值
    unit VARCHAR(20),                          -- 单位
    value_table TEXT,                          -- 值表（JSON格式，如{"0":"关闭","1":"开启","2":"故障"}）
    description TEXT,                          -- 描述
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (matrix_id) REFERENCES can_matrices(id)
);
```

```sql
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (matrix_id) REFERENCES playback_matrices(id) ON DELETE CASCADE,
    FOREIGN KEY (source_slot_mapping_id) REFERENCES audio_source_slot_mappings(id) ON DELETE CASCADE,
    UNIQUE(matrix_id, source_slot_mapping_id)
);
```

#### 13. 工况播放矩阵表 (playback_matrix_conditions)
```sql
-- 各种工况下的扬声器出声配置
-- 支持多个SLOT同时开启，如媒体音立体声的SLOT1和SLOT2
CREATE TABLE playback_matrix_conditions (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    matrix_id INTEGER NOT NULL,                -- 所属播放矩阵
    source_type_id INTEGER NOT NULL,           -- 音源类型
    source_status VARCHAR(50),                 -- 音源状态
    slot_ids TEXT NOT NULL,                    -- SLOT列表（JSON数组，如[1,2]表示SLOT1和SLOT2同时开启）
    headrest_mode VARCHAR(50),                 -- 头枕模式
    sound_field_focus VARCHAR(50),             -- 声场聚焦
    speaker_mapping TEXT,                      -- 出声扬声器映射（JSON数组）
    description TEXT,                          -- 描述
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (matrix_id) REFERENCES playback_matrices(id) ON DELETE CASCADE,
    FOREIGN KEY (source_type_id) REFERENCES audio_source_types(id) ON DELETE CASCADE
);
```

5. 数据验证（显示错误和警告）
6. 确认导入
7. 导入结果统计

**导入历史**
- 导入记录列表（时间、类型、文件名、状态、成功/失败数）
- 导入详情（错误日志）

## 关键特性

### 1. 完整的数据关联体系

```
需求 ──┬──► 测试用例 ──► 测试场景 ──► 测试脚本
       │      │
       │      └──► 适用车型
       │
       └──► 需求车型详情

平台 ──► 车型 ──┬──► 需求车型详情
               ├──► 用例车型关联
               └──► 车型配置 ──┬──► 扬声器通道映射
                              └──► 播放矩阵

音源类型 ──► 音源-SLOT映射 ◄── A2B SLOT
播放矩阵 ──┬──► 基本矩阵
           └──► 工况矩阵
```

### 2. 多车型管理

**平台分层管理**
- 一级：平台（Platform-A, Platform-B, ...）
- 二级：车型（Model-1, Model-2, ...）

**需求车型矩阵视图**
- 横轴：按平台分组的车型列表
- 纵轴：需求列表
- 交叉点：适用性标记 + 实现细节

**批量编辑功能**
- 支持按平台批量选择车型
- 支持批量设置需求/用例的车型适用性
- 支持批量导入车型关联数据

### 3. Gauge测试框架集成

**三层结构**
- Spec文件 → 测试脚本（test_scripts）
- Scenario → 测试场景（test_scenarios）
- 测试用例 → 测试用例（test_cases）

**数据驱动支持**
- 场景可标记为数据驱动（is_data_driven = true）
- 数据表存储在data_table字段（JSON格式）
- 每行数据对应一个测试用例（通过data_row_index关联）

**场景-用例关联**
- 非数据驱动：1个场景 → 1个用例
- 数据驱动：1个场景 → N个用例（每行数据一个）

### 4. 音源与SLOT多对多映射

**独立管理**
- 音源类型独立定义（媒体音、杜比、蓝牙电话、AVAS、ANC等）
- A2B SLOT独立定义（SLOT1-SLOT26等）
- 通过映射表建立多对多关系

**灵活配置**
- 一个音源可使用多个SLOT（如杜比7.1.4使用SLOT1-12）
- 一个SLOT可被多个音源使用（如SLOT1被媒体音和杜比共用）
- 每个映射可指定SLOT角色（左声道、右声道、中置等）

### 5. 播放矩阵双层设计

**基本播放矩阵（总览）**
- 展示每种音源的基础SLOT和扬声器配置
- 作为播放矩阵的概览视图

**工况播放矩阵（详细）**
- 按条件配置：音源状态、头枕模式、声场聚焦
- 支持多SLOT同时开启（如立体声的SLOT1+2）
- 精确控制每种工况下的扬声器出声情况

### 6. 数据导入导出

**Excel导入**
- 支持需求、用例、CAN矩阵、扬声器映射、播放矩阵的导入
- 导入预览和数据验证
- 字段映射配置
- 导入历史记录

**模板下载**
- 提供标准Excel导入模板
- 模板包含填写说明和示例数据

### 7. 版本和状态管理

**版本管理**
- CAN矩阵：版本号字段
- 测试脚本：版本号字段
- 播放矩阵：通过名称区分版本

**状态管理**
- 需求：draft → approved → implemented → verified

## 安全考虑

### 1. 认证和授权

**JWT认证流程**
```
用户登录 → 验证凭据 → 生成JWT Token → 返回Token
    ↓
后续请求 → 携带Token → 验证Token → 处理请求
```

**基于角色的访问控制（RBAC）**
```python
# 角色定义
ROLES = {
    'admin': ['all'],           # 管理员：全部权限
    'engineer': ['read', 'write'],  # 工程师：读写权限
    'viewer': ['read']          # 查看者：只读权限
}
```

**权限粒度**
- 模块级权限：需求管理、用例管理、脚本管理等

- 存储过程使用参数化调用

**XSS防护**

- Content-Security-Policy头部配置

**CSRF防护**

- 关键操作要求二次确认

### C. 常用SQL查询示例

#### 查询需求的车型覆盖情况
```sql
SELECT 
    r.req_id,
    r.title,
    COUNT(CASE WHEN rvd.is_applicable = 1 THEN 1 END) as applicable_count,
    COUNT(vm.id) as total_vehicles
FROM requirements r
CROSS JOIN vehicle_models vm
LEFT JOIN requirement_vehicle_details rvd 
    ON rvd.requirement_id = r.id AND rvd.vehicle_model_id = vm.id
GROUP BY r.id;
```

#### 查询测试用例的场景关联
```sql
SELECT 
    tc.case_id,
    tc.title as case_title,
    ts.scenario_name,
    tst.spec_name,
    tst.file_path
FROM test_cases tc
LEFT JOIN test_scenarios ts ON ts.id = tc.scenario_id
LEFT JOIN test_scripts tst ON tst.id = ts.script_id;
```

#### 查询播放矩阵详情
```sql
SELECT 
    pm.name as matrix_name,
    vc.vehicle_model,
    ast.source_name,
    pmc.source_status,
    pmc.slot_ids,
    pmc.headrest_mode,
    pmc.sound_field_focus,
    pmc.speaker_mapping
FROM playback_matrices pm
JOIN vehicle_configs vc ON vc.id = pm.config_id
JOIN playback_matrix_conditions pmc ON pmc.matrix_id = pm.id
JOIN audio_source_types ast ON ast.id = pmc.source_type_id
ORDER BY pm.id, ast.source_name;
```

#### 查询音源的SLOT配置
```sql
SELECT 
    ast.source_name,
    ast.channel_count,
    GROUP_CONCAT(
        ab.slot_number || '(' || assm.slot_role || ')', 
        ', '
    ) as slots
FROM audio_source_types ast
JOIN audio_source_slot_mappings assm ON assm.source_type_id = ast.id
JOIN a2b_slots ab ON ab.id = assm.slot_id
GROUP BY ast.id
ORDER BY ast.source_name;
```
